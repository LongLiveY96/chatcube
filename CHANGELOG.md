# Changelog

## v1.0.10 (2026-03-20)

这一版不是修修补补，而是把工具链、列表渲染和服务商兼容层一起收了一轮：MCP 工具中心正式落地，聊天内工具调用状态更稳定，会话/消息列表的数据源与复用链路也更清晰，整体可维护性比之前高一个量级。

### ✨ 新增 / 改进

- 新增 MCP 服务接入和工具中心主流程，统一承接内置工具与远端 MCP 工具
- 聊天页补齐工具选择、审批和执行相关 UI，工具栏 pills 与工具调用卡片展示更稳定
- 推理能力配置从服务商策略中解耦，模型编辑页和聊天页按模型真实能力展示 reasoning 相关入口
- 会话列表和消息列表引入更清晰的数据源与列表项结构，长列表滚动和复用表现更稳
- 隐私政策改为应用内二级页面承载，避免跳转外部浏览器打断当前设置流

### 🐛 修复

- 修复工具事件、工具状态在发送流程、落库恢复和消息 UI 之间不同步的问题
- 修复服务商兼容判断、模型获取和分享预览链路中的一批分叉问题，provider 图标和分享内容更稳定
- 修复首页 session list 更新链路里未读角标和多处冗余状态带来的刷新噪音
- 修复消息列表复用、滚动联动和部分工具消息卡片在复杂场景下的渲染异常
- 修复等待 AI 回复时 typing indicator 三点动画丢失的问题
- 修复首页搜索展开后输入框不自动聚焦、软键盘不弹出的交互回归
- 修复系统提示词、显示与外观、工具服务等设置页底部内容与 compact tabs 贴边甚至被遮挡的问题

### 🔧 内部优化

- 新增 `ProviderCompatibilityRegistry`、`McpService`、列表 DataSource 等基础设施，明显减轻 `ChatPage` 和 `Index` 的状态负担
- 重构聊天工具相关 sheet 组件，减少 `ChatPage` 膨胀，降低后续继续堆特殊分支的风险
- 补充 `RefactorGuard.test.ets` 和 `ohosTest` 入口，为本轮结构调整加上基础回归边界

## v1.0.9 (2026-03-16)

这一版主要把服务商配置、聊天发送稳定性和提示反馈再收紧一轮：服务商详情页更顺手，附件重发和返回提示更可靠，MiniMax 兼容链路也补齐了。

### ✨ 新增 / 改进

- 服务商详情页的标题与地址信息对齐优化，编辑和查看状态更统一
- MiniMax 预置服务商升级到最新兼容地址，连接测试与模型获取流程更顺
- 服务商测试连接、获取模型失败时优先展示真实错误信息，排查更直接

### 🐛 修复

- 修复会话重新打开时总是回到全局当前模型的问题，改为优先恢复该会话上次使用的模型
- 修复服务商详情页标题首字显示不完整、URL 摘要未对齐的问题
- 修复 Provider Detail 在不同宿主模式下的返回链路和若干交互细节
- 修复附件在部分发送链路下重发不完整的问题，补齐请求附件准备流程
- 修复应用内提示反馈混用导致的返回体验不一致问题，普通提示回归 toast，操作型通知保留 snackbar
- 修复 MiniMax 兼容判断过宽的问题，避免用户修改网关后仍被误判为 MiniMax 专用链路

### 🔧 内部优化

- 收拢服务商兼容配置和错误提取逻辑，减少重复判断
- 清理一批未使用字符串资源，减轻资源噪音
- 继续收敛模型管理、聊天页和设置页的状态同步与显示细节

## v1.0.8 (2026-03-13)

这一版主要把模型管理和聊天列表这两条高频路径收稳了：服务商详情不再莫名收起，模型选择更清楚，导入导出也少了很多笨重的文件复制逻辑。

### ✨ 新增 / 改进

- 服务商模型选择器改成按品牌快速跳转，已选状态和分组展示更清晰
- 聊天里的模型选择器分组显示更稳定，不再偶发串到错误服务商名下
- 会话列表和消息列表补上更稳的主题刷新与缓存策略，长列表滚动更顺

### 🐛 修复

- 修复服务商管理里连续关闭再打开详情 sheet 时偶发自动收起的问题
- 修复聊天消息和会话卡片在主题切换、消息复用时状态串位的问题
- 修复导入导出附件时的大文件复制与会话摘要回填逻辑，降低失败概率
- 修复智谱模型分组命名和部分模型能力匹配覆盖自定义名称的问题

### 🔧 内部优化

- 导入导出统一改用 `fileIo.copyFile`，去掉多处手搓二进制拷贝
- 数据库消息导入支持延后刷新会话摘要，减少重复计算
- README 文案里的“微信赞赏码”统一改成“微信打赏码”

## v1.0.5 (2026-03-06)

这一版主要解决数学公式显示和 AI 画图的问题，顺便修了个烦人的 bug。

### ✨ 新增 / 改进

- 聊天标题支持显示数学公式了（AI 起的标题带公式不再显示乱码）
- 会话列表卡片里的标题也能正确显示数学公式
- 消息气泡里的数学公式兼容更多模型格式（OpenAI 等用 `\[...\]` 写法的也能渲染了）
- 模型编辑页新增「图片生成」能力标签，可手动标记或自动识别
- 模型自动匹配更智能：中转站给模型加后缀也能正确识别能力了

### 🐛 修复

- 修复聊天卡片列表页长按功能失效的问题
- 修复部分 AI 画图模型返回图片链接（而非 base64）时图片无法显示的问题

### 🔧 内部优化

- AI 生成图片的保存逻辑统一抽取，消除了 4 处重复代码

## v1.0.4 (2026-03-04)

这一版主打「更顺手 + 更稳定」：模型学会分组列队、输出不抢你滚动、设置弹窗更可靠。

### ✨ 新增 / 改进

- 获取模型支持分组显示（找模型更快）
- 支持聊天字体大小调节（看得更舒服）
- 对话列表更紧凑：左右留白优化、左滑操作区更克制，FAB 更好点

### 🧠 体验优化

- 模型输出内容时不再影响用户上下滑动查看内容
- 服务商界面键盘避让，输入不再被挡住

### 🛠 稳定性与修复

- 设置页弹窗逻辑优化，更加稳定可靠
- 优化异常情况报错提示，更清晰可读
- 优化聊天界面大量消息时 jump 到底的逻辑，不再闪退
