# Changelog

## v1.1.3 (2026-05-09)

这一版主要为上架和高频聊天体验收口：接入华为账号登录并补齐 AI 生成标识；把模型侧高级生成参数迁移到助手设置；完善附件、Mermaid、OpenAI Responses、联网搜索和分享链路；同时修复多会话并发、Markdown 图片保存、分享图片内容丢失等问题。

### ✨ 新增 / 优化

- 接入华为账号登录，并在对话页底部补充 AI 生成标识，为应用上架合规做准备
- 将温度、Top K 等高级生成参数从模型配置迁移到助手设置，按助手统一管理个性化生成偏好
- 优化附件上传与解析能力，支持更多类型附件读取正文并注入聊天上下文
- 宽内容消息气泡自适应放宽，表格、代码块和 HTML 等内容更易阅读
- 支持 Mermaid 流程图预览，`mermaid` 代码块可直接查看图形、源码、复制和全屏预览，Markdown 预览能力进一步补齐
- 关于页新增邀测地址入口，README 邀测地址改为活码，分享应用更方便
- OpenAI Responses 协议补齐工具调用、推理摘要、历史编码和分享导出等结构
- 联网搜索链路更稳妥，避免工具调用预算和异常分支导致回答中断

### 🐛 修复

- 修复助手回复中的 Markdown 图片进入预览后无法保存到相册的问题
- 修复多会话并发发送、手动搜索和后台任务更新时状态可能串到当前页面的问题
- 修复助手消息头像不跟随消息实际使用模型的问题
- 修复助手 Tab 列表底部被 FAB 挡住的问题
- 修复 OpenAI 兼容协议在推理关闭时仍下发 `reasoning_effort=none` 的兼容性问题
- 修复 Responses 历史消息编码、推理摘要请求和推理标识展示的一批细节问题
- 修复图片分享时部分对话内容丢失的问题

## v1.1.0 (2026-04-30)

这一版从 `1.0.13` 继续往前收了一轮主体验：新增助手模块，把多助手管理、自定义助手头像、默认模型和系统提示词这些能力集中起来；聊天、设置与模型管理等高频页面统一换成更贴近 HarmonyOS 的 HDS 材质与渐变模糊；主题配色也重新整理，新增 Notion 主题。推理控制按不同服务商能力做了更细的适配，导入导出策略同步调整为恢复后可直接使用，为后续 WebDAV 全量同步打基础。

### ✨ 新增 / 改进

- 新增助手模块，支持在首页侧边入口管理多个助手，并按助手查看会话、快速创建专属对话
- 助手模块内支持自定义助手头像，可使用文字、图片链接、QQ 头像、本地图库图片和默认模型图标，列表页、详情页和编辑页的头像展示尺寸更统一
- 助手模块支持默认模型、系统提示词、温度覆盖等信息，默认助手和自定义助手的编辑、删除、排序与会话关联更清楚
- 聊天卡片页顶部接入 `HdsNavigation` 材质导航，搜索、收藏、标题栏与顶部渐变模糊范围重新整理，滚动时顶部质感更稳定
- 聊天 Tab 收敛为单 `HdsNavigation` 自适应分栏，大屏空态、返回路径和列表 / 详情切换更稳定
- 设置、模型管理、导入导出、关于、通知等二级页统一接入 HDS 顶部模糊效果，页面切换时材质表现更一致
- 主题配色重新整理，新增 Notion 极简主题，默认暖纸、Claude、ChatGPT、Violet、Mint 等主题的消息、卡片、Markdown 和桌面小组件配色更协调
- 首页筛选交互收敛，弱化「全部 / 收藏」Tab 的打扰，搜索展开与返回后的标题栏状态更稳
- 底部 / 侧边 Tab 补齐点击即时缩放反馈，助手 Tab 的选中态图标染色与对话页保持一致
- 推理控制按模型和服务商能力细分，补齐自动、低、中、高、最高等推理级别，并兼容更多 OpenAI 兼容服务商策略
- 新增小米 MiMo 预置服务商，并补齐 MiMo thinking / reasoning 兼容判断
- 导出数据默认包含 API Key、搜索服务和 MCP 连接配置，导入后可直接使用，同时在导出页明确提示备份文件需要保存到可信位置

### 🐛 修复

- 修复助手头像编辑后列表、设置页和对话入口不及时刷新的问题
- 修复聊天卡片页搜索展开后新建对话、返回列表时顶部搜索和收藏入口丢失的问题
- 修复搜索展开动画在状态恢复后缺失的问题
- 修复聊天卡片页左上角助手头像与助手设置页头像尺寸不一致的问题
- 修复会话列表多选、筛选和收藏状态切换时部分临时状态未正确回收的问题
- 修复推理级别在部分服务商和模型组合下可用性判断不准确的问题
- 修复部分组件仍读取旧主题色导致切换主题后颜色不一致的问题
- 修复后台任务完成后通知未及时清理的问题

### 🔧 内部优化

- 新增 `AssistantService`、助手头像缓存与助手管理页，减少首页对助手数据和编辑状态的直接耦合
- 补充助手、头像、服务商、偏好设置在导入导出中的结构，为 WebDAV 全量快照同步预留统一备份基础
- 抽出 HDS 顶部模糊、点光效果和聊天顶部导航相关状态，减少导航栏、搜索栏、筛选栏之间的布局联动成本
- 重整主题色服务与全局 UI 状态，减少主题资源、页面硬编码颜色和桌面小组件之间的同步成本
- 扩展 `ProviderCompatibilityRegistry`，把不同服务商的推理请求策略集中维护，降低发送链路里的特殊判断

- 聊天标题支持显示数学公式了（AI 起的标题带公式不再显示乱码）
- 会话列表卡片里的标题也能正确显示数学公式
- 消息气泡里的数学公式兼容更多模型格式（OpenAI 等用 `\[...\]` 写法的也能渲染了）
- 模型编辑页新增「图片生成」能力标签，可手动标记或自动识别
- 模型自动匹配更智能：中转站给模型加后缀也能正确识别能力了

### 🐛 修复

- 修复聊天卡片列表页长按功能失效的问题
- 修复部分 AI 画图模型返回图片链接（而非 base64）时图片无法显示的问题

### 🔧 内部优化

- AI 生成图片的保存逻辑统一抽取，消除了 4 处重复代码
